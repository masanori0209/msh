# 競合分析

## 比較サマリー

| 観点 | Bash | Zsh | Fish | **msh（目標）** |
|---|---|---|---|---|
| 実装言語 | C | C | C++ | **Rust** |
| 起動速度 | ◎ 最速クラス | ○ | ○ | ◎ 目標 |
| メモリ | ◎ ~2MB | △ プラグインで増大 | ○ ~5MB | ◎ <5MB |
| スクリプト互換 | ◎ デファクト | ○ Bash 互換 | △ 独自構文 | **○ Zsh 同等（目標）** |
| 対話 UX | △ 素朴 | ○ Oh-My-Zsh 依存 | ◎ 業界最高クラス | **◎ 全競合超え（目標）** |
| 設定の複雑さ | ○ シンプル | △ 複雑化しやすい | ○ 自動 | ◎ 1 ファイル |
| 拡張性 | △ 関数のみ | ◎ プラグイン豊富 | ○ 関数 + テーマ | ○ Phase 6 |
| メンテナ | GNU / 各種 | コミュニティ | コミュニティ | 自前 |

---

## Bash

### 強み
- POSIX 互換、どこでも動く
- 起動が最速、メモリ最小
- 30 年以上のエコシステム

### 弱み
- 対話 UX が古い（補完・ハイライトは別途設定）
- エラーメッセージが分かりにくい
- 配列・連想配列など言語機能が貧弱

### msh への示唆
- **Zsh 並みの互換性**を目指し、移行コストを下げる
- 起動速度・メモリは Bash をベンチマークの下限とする

---

## Zsh

### 強み
- 強力な補完・グロブ・スペル修正
- Oh-My-Zsh / Prezto 等の巨大エコシステム
- Bash との高い互換性

### 弱み
- 設定ファイルが肥大化しやすい（`.zshrc` 数百行が普通）
- プラグイン込みで起動が遅く、メモリも増える
- 「設定しないと Bash と大差ない」問題

### msh への示唆
- **設定レスで Zsh+Oh-My-Zsh 相当の UX** をデフォルト提供
- **Zsh 並みの `.bashrc` 互換**で移行障壁を下げる

---

## Fish

### 強み
- 対話 UX のゴールドスタンダード
  - シンタックスハイライト
  - インライン autosuggestion
  - Web ベースの設定 UI
- 初心者に優しいエラーメッセージ
- 起動時に自動設定（`fish_config`）

### 弱み
- Bash スクリプト非互換（`bash script.sh` が必要）
- C++ 実装で Rust 製競合より最適化余地がある
- 一部 POSIX 非互換（`&&` / `||` 等の歴史的問題は解消済み）

### msh への示唆
- **Fish の UX をベースライン**として、**全競合を上回る**初心者体験を目指す
- **Fish 非互換の弱点を Zsh 並みの互換性**で補う

## msh のポジショニング

```
        高 UX
          │
    ★ msh（目標: Fish 超え + 初心者ファースト）
          │
    Fish  │
          │
    Zsh   │  ← 互換性のベンチマーク
          │
    Bash  │
          │
        低 UX ────────────────── 高 互換性
```

**短期**: Zsh 並みの互換 × Fish 超えの UX  
**中期**: Zsh ユーザー + 初心者のデフォルトシェル候補  
**長期**: 互換性 L4（80%+ `.bashrc`）+ UX で全競合を差別化
