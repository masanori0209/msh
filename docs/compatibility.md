# 互換性方針

Zsh 同等の移行しやすさを目標とした、段階的互換ロードマップ。

---

## 目標の定義

| 用語 | 意味 |
|---|---|
| **Bash 100% 互換** | POSIX + Bash 5.x の全機能を再現 | ❌ 非目標 |
| **Zsh 同等** | 日常の対話 + 一般的な `.bashrc` / スクリプトの大部分が動く | ✅ 目標 |
| **移行しやすさ** | 既存設定を `source` し、最小変更で日常使いできる | ✅ 目標 |

---

## 互換性レベル（L1–L5）

### L1: 対話の基本 ✅ 完了（v0.2）

- [x] パイプライン、リダイレクト
- [x] 環境変数展開 `$VAR` `${VAR}`
- [x] 組み込み: cd, export, alias, source, echo, exit
- [x] バックグラウンド `&`、Ctrl+C
- [x] glob `*` `?`

### L2: シェル構文（Phase 5） ✅

- [x] コマンドチェイン: `&&` `||` `;`
- [x] コマンド置換: `$()` `` ` `` ``
- [x] クォート中の展開ルール（Bash 準拠・単一引用符リテラル）
- [x] 終了コード `$?`
- [x] ヒアドキュメント `<<EOF`（基本）

### L3: スクリプト言語（Phase 5–6）

- [x] 関数定義 `name() { ... }`
- [x] 制御構造: `if` / `for` / `while` / `case`
- [x] 配列（インデックス、`"${arr[@]}"`）
- [x] ローカル変数 `local`
- [x] `return` / `break` / `continue`

### L4: 移行体験（Phase 5） ✅

- [x] `.bashrc` / `.zshrc` / `.mshrc` の読み込み順序
- [x] `msh --compat zsh` / `--compat bash` フラグ
- [x] 未対応構文の明示エラー（行番号 + 回避策）
- [x] 移行ガイド（docs/migration.md）
- [x] 互換性テストスイート（代表 `.bashrc` スニペット）

### L4 移行体験で追加実装（v0.7.1）

- [x] `[[ ]]` 条件式（単項 `-z/-n/-e/-f/-d`、文字列 `==`/`!=`、数値 `-eq` 等、`!` 否定）
- [x] プレーン代入 `VAR=value`（`export` なし）
- [x] `A && B || C` の短絡チェイン
- [x] 空クォート保持（`[[ -n "$X" ]]` の正しい判定）
- [x] `set -e`（errexit）/ `set -u`（nounset）
- [x] `$PIPESTATUS[n]`
- [x] 算術展開 `$(( ))`（`+ - * / %`、括弧、変数）

### L5 高度構文で実装（v0.7.2）

- [x] 連想配列 `declare -A` / `typeset -A`
  - `m[key]=value` 代入 / `${m[key]}` 取得 / 添字の変数展開 `${m[$k]}`
  - `${!m[@]}`（キー）/ `${#m[@]}`（要素数）/ `${m[@]}`（全値）
- [x] 添字付き配列の要素代入 `arr[2]=z`（スパース埋め）

### L5 高度構文で実装（v0.7.3）

- [x] パラメータ展開の高度形
  - `${var:-default}` / `${var:+alt}` / `${var:?msg}`（`:` 有無で「未設定」か「未設定または空」を切替）
  - `${var#pat}` / `${var##pat}` / `${var%pat}` / `${var%%pat}`（glob 前後除去）
  - `${var/pat/rep}` / `${var//pat/rep}`（最初/全置換）
  - `${var^^}` / `${var^}` / `${var,,}` / `${var,}`（大小変換）
  - `${var:offset:length}`（部分文字列・負オフセット可）/ `${#var}`（文字長）/ `${!var}`（間接参照）
- [x] `$RANDOM`（依存なしの簡易 PRNG）

### L5: 未対応（必要時は bash 委譲）

- 連想配列の複合初期化 `declare -A m=([k]=v)`
- `${var:=default}` の永続代入（値は返すが変数へは代入しない）
- Process substitution `<()` `>()`
- Zsh 固有: 拡張 glob `**(/)`
- 算術の `**`・三項・ビット演算（基本四則のみ対応）
- `$SECONDS` / `$LINENO`

> 現状の自動計測スコアは [compat-score.md](./compat-score.md) を参照（`scripts/compat-score.sh` で再生成）。

## 競合との互換性比較

| 機能 | Bash | Zsh | Fish | msh 目標 |
|---|---|---|---|---|
| `.bashrc` そのまま | ◎ | ○ | ✗ | ○ (L4, 実測 91%) |
| `&&` `\|\|` | ◎ | ◎ | ◎ | ◎ (L2) |
| `$()` 置換 | ◎ | ◎ | ◎ | ○ (L2) |
| `[[ ]]` 条件式 | ◎ | ◎ | ✗ | ○ (L4 基本) |
| 算術 `$(( ))` | ◎ | ◎ | △ | ○ (L4 四則) |
| 関数・if/for | ◎ | ◎ | △ | ○ (L3) |
| 配列（インデックス） | ◎ | ◎ | ✗ | ○ (L3) |
| 連想配列 `declare -A` | ◎ | ◎ | ✗ | ○ (L5) |
| パイプ・リダイレクト | ◎ | ◎ | ◎ | ◎ (L1) |
| エイリアス | ◎ | ◎ | ◎ | ◎ (L1) |

---

## 未対応時の方針

1. **明確なエラー** — 「この構文は未対応です。回避: …」
2. **bash 委譲** — `bash -c '...'` または `msh` 内から自動フォールバック（将来検討）
3. **ドキュメント** — 互換性マトリクスを公開し、期待値を管理

## 互換性 KPI

| 指標 | Phase 5 完了時 |
|---|---|
| 代表 `.bashrc` スニペット通过率 | ≥ 80% |
| 代表 CI スクリプト（git hook 等）通过率 | ≥ 80% |
| 未対応構文のエラーに回避策記載率 | 100% |

代表スニペット例:

```bash
export PATH="$HOME/bin:$PATH"
alias ll='ls -la'
[ -f ~/.bashrc.local ] && source ~/.bashrc.local
cd /tmp && git status
for f in *.txt; do echo "$f"; done
```
