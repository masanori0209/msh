> 計測日: 2026-06-28

## エグゼクティブサマリー

msh は **「Fish 級の対話 UX × Zsh 移行を意識した互換性 × Rust 製の軽量バイナリ」** を軸に、v0.7.3 時点で次の位置にいる。

| 軸 | 現状 | 変化 (v0.7.2→v0.7.3) |
|---|---|---|
| 互換 | **98% (41/42)** | パラメータ展開高度形（残りはプロセス置換のみ・非目標） |
| バイナリ | **816 KB** | 2.0 MB → −60% |
| 起動 RSS | **6.22 MB** | 6.39 MB → −3%（bash 比 ~3.1×、8MB KPI 内） |
| テスト | **106/106** | +15 件 |
| 起動速度 | zsh **同等〜やや遅** | 交互計測で誤差範囲 |

**現在地（一言）**: Fish に近い対話 UX と **816KB 単一バイナリ**で差別化しつつ、互換 **98%**（対象範囲は実質全通過、唯一の不合格はプロセス置換で非目標）で **日常移行は実用域**。RSS は bash 比 ~3 倍（Rust ランタイム下限・**8MB KPI 達成**）。本番デフォルトシェル化の残りは **初回 GitHub Release 実行** と **WASM プラグイン PoC**。

## 性能ベンチマーク

`MSH_SKIP_RC=1`、release ビルド、macOS ローカル（2026-06-28 計測）。

### 起動・メモリ (`-c exit`)

| シェル | ウォーム起動 (交互 200 回・中央値) | max RSS |
|---|---|---|
| bash | **4.56 ms** | **1.89 MB** |
| zsh | **5.97 ms** | **2.30 MB** |
| **msh** | **6.58 ms** | **6.22 MB** |

> 起動は **zsh と同等帯（誤差・負荷で前後）**。bash が最速。絶対値はシステム負荷で変動するため交互計測を推奨。

### バイナリ・依存（v0.7.2 省メモリ化）

| 指標 | v0.7.0 | v0.7.2 | 差分 |
|---|---|---|---|
| バイナリ | 2.0 MB | **816 KB** | **−60%** |
| 起動 RSS | 6.39 MB | **6.22 MB** | −3% |
| 依存 | 124 | **111** | −13 |

主因: regex 除去＋Ctrl+R 自前実装、serde 廃止、rustyline trim、`panic=abort`。詳細: [memory-optimization.md](./memory-optimization.md)。

Phase 4 目標（RSS < 8 MB, 起動 < 30 ms）: ✅ 達成。

## 競合比較（現状スコア）

5 段階: ◎ 4 / ○ 3 / △ 2 / ✗ 1 / — 0

| 観点 | Bash | Zsh | Fish | **msh v0.7.2** | 目標 |
|---|---|---|---|---|---|
| 起動速度 | ◎ | ○ | ○ | **○**（zsh 同等帯） | ◎ |
| メモリ | ◎ | △ | ○ | **○**（816KB バイナリ / RSS ~3× bash） | ◎ |
| スクリプト互換 | ◎ | ◎ | △ | **○+**（98%） | ○ |
| 対話 UX | △ | ○ | ◎ | **○+** | ◎ |
| 設定の簡潔さ | ○ | △ | ○ | **◎** | ◎ |
| 拡張性 | △ | ◎ | ○ | **△** | ○ |

### msh が優位な点

- **816KB 単一バイナリ** — regex/serde なし、Homebrew/Release 配布向き
- **設定 1 ファイル** (`config.toml`) + UX 機能デフォルト ON
- **ファジー補完 + 説明** + 日英エラー + オンボーディング
- **Ctrl+R 大小文字無視**（依存追加なしの自前実装）
- **未対応構文の明示エラー** + 回避策

### msh が劣る点

- **RSS**: bash 比 ~3.1 倍（Rust ランタイムの構造的下限・8MB KPI 内）
- **互換**: プロセス置換 `<()`、`${var:=}` の永続代入、`$SECONDS`/`$LINENO`
- **エコシステム**: 初回 Release 未実行 / WASM プラグイン PoC 未着手
- **Fish 比**: Web 設定 UI、autosuggest ソースの豊富さ

## ポジショニング図

```
        高 UX
          │
    Fish ─┤
          │        ★ msh v0.7.2
          │           ↗ UX + 816KB バイナリ
    Zsh ──┤
          │
    Bash ─┤
          │
        低 UX ────────────────── 高 互換性
              Bash/Zsh          Fish
```

**短期目標との距離**

| 目標 | 進捗 |
|---|---|
| Fish 超え UX | **~78%** — コアは揃う、polish / 設定 UI に及ばず |
| Zsh 並み互換 | **~94%** — 98% スコア + dotfiles 必須 100% + パラメータ展開 |
| 起動・メモリ差別化 | **~80%** — 816KB は突出、RSS bash 超えは構造的下限 |
